1. All Angular projects starts with the "index.html" and "app.ts"

  • index.html is a base html file which calls for "< app-root></app-root>"
  • Then the angular looks for app-root decorator in components
  • Now Main.ts comes in to picture. The it registers the app component which has the app-root decorator
  • Finaly, App component's HTML will get loaded

But we can also change the initial loading root component.
we need to create a new component under any name you desire eg: "main-component", then we need to change the "index.html" call and register this new component in "main.ts"

ie: From "< app-root></app-root>" To "<main-component></main-component>"
So, basically all the content that you are readin now is loaded from main-component and not from app-root component.

2. Interpolation

String interpolation is a simple way to use a variable from a .ts file to its .html file in a component
i've declared a variable "name" in the main-component.ts file and to use it we need to use double curly bracis in html file.
Eg : {{name}}

we can also invloke a method using interpolation! here i've declared a mehtod named "sample_method()" in the .ts file and i can call this from html using same double curly bracis
Eg: {{sample_method()}}

In this case, i can use property binding with "[]" in html input tag binding with the .ts variable name "isDisabled", now angular takes the value from .ts file and uses it against the input tag in HTML file

  • Property Binding

    • So, The above input fied is disabled because the "isDisabled" variable in .ts is set to true and attached to html input tag like this : [disabled]="isDisabled"

  • Style Binding

6. Directives
Directives are basically classes in Angular. It can be defined as a class which can alter the behaviour of the DOM.
In this case, we have few types of Directives

  • Components
  • Structural
  • Attribute
All these have few inbuilt directives and we can also create custom ones.
